# v1.8.0 Consolidation Implementation Plan (Phase 2)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement Phase 2 of `docs/full-review-2026-07-02.md` — consolidate redundant GUI/features (sections D1–D5), delete dead code (section E), and clean the repo, shipping as app v1.8.0 / extension v4.47. The app gets SMALLER: fewer buttons, fewer permissions, less code.

**Architecture:** Web UI consolidations happen inside `web/index.html` following its existing patterns (HTML-string render functions, inline handlers, `persistCards()`, `toast()`). Extension work removes the legacy bridge-driving layer entirely — the service worker (v4.40+ unified driver) is the only capture driver. No new frameworks, no bundler, no schema changes.

**Tech Stack:** Vanilla JS renderer, Chrome MV3 extension, plain-Node tests (`node tests/<file>.test.js`, suite `npm test`).

**Plan-style note (pre-approved deviation from the plan-format rules):** renderer/extension UI tasks specify exact behavioral requirements, acceptance criteria, and the inventory of what to remove/merge — not verbatim replacement code — because the implementer must read the current 4,500-line index.html state first. This matched-latitude approach was validated across Phase 1 (Tasks 6–8). TDD applies wherever a harness exists (extension pure code, new pure web modules); index.html changes are verified by the repo's syntax gate + source assertions + reviewer trace, as in Phase 1.

Remove message handlers used only by bridge.js (`getQueue`, `clearQueue`, `captureRequest` forwarding, `captureOneTab` IF its only sender was bridge.js — verify each by grepping senders across the extension before removing; popup.js and capture-core.js senders must keep working).

- [ ] Tests: delete/replace bridge tests. NEW `tests/ext-sw-driver.test.js` with source assertions locking the new invariants: no `APP_TAB_URLS`/`localhostTabOpen` references remain; `pollBatchState`/`pollCaptureRequest` contain no defer-on-tab branch; manifest has NO localhost matches.
